# P-2026-001: 数据资产等级划分

## 目标

对数据仓库中用户指定的数据应用/数据查询进行数据资产等级划分。每个数据应用/查询，向上追溯完整血缘链路（至 ODS 层），对链路上所有涉及的表给出等级划分建议，输出为 markdown 文件。

## 背景

数据资产等级划分是企业数据治理的基础。通过标识仓库中每张表的重要性等级（P0-P4），可以：
- 指导数据质量保障资源的分配优先级
- 为调度链路监控和告警策略提供依据
- 支撑数据生命周期管理和成本优化

## 工作模式

用户指定 DML/应用 SQL 文件路径，并说明：
1. **SQL 文件路径**：`Application/` 或 `starrocks/` 下的 SQL 文件路径
2. **重要性等级**：该数据应用/查询本身的等级（P0-P4）
3. **应用场景**：该数据应用/查询的业务用途

Agent 收到指定后：
1. 读取目标 SQL 文件，识别引用的数仓表
2. 向上追溯血缘链路至 ODS 层
3. 对链路上每张表给出等级划分建议
4. 输出分析报告到 `workspace/` 目录

## 等级传导规则

采用「下游辐射上游」规则：
- 目标应用/查询为 P0 → 血缘链路上所有上游表最低 P1
- 目标应用/查询为 P1 → 血缘链路上所有上游表最低 P2
- 目标应用/查询为 P2 → 血缘链路上所有上游表最低 P3
- 目标应用/查询为 P3 → 血缘链路上所有上游表最低 P4

当一张表出现在多个数据应用/查询的血缘链路上时，取其等级的最高者（P0 最高，P4 最低）。

## 输出规范

每个数据应用/查询在 `workspace/` 下生成一个 markdown 文件，命名格式：`{目标表/查询简称}_资产等级分析.md`。

文件内容包含：
- **目标对象信息**：表名、所属层级、等级、应用场景
- **血缘链路图**：从目标表到 ODS 的完整链路
- **逐表分析**：每张表的层级、表名、建议等级、判定理由
- **汇总表**：链路所有表的等级一览
