## 逐表等级划分

### 应用层 (Application)

| 表名 | 层级 | 建议等级 | 判定理由 |
|------|------|----------|----------|
| `模板&剧维度基建上限.sql` | Application | **P0** | 用户指定。市场部广告优化师核心工具，分析创编广告基建花费与达标情况，直接支撑广告投放决策 |

### ADS 层

| 表名 | 建议等级 | 判定理由 |
|------|----------|----------|
| `ads_srsv_bi_ad_optimizer_template_target_data_1` | **P0** | 该应用直接消费的 ADS 表，承载模板维度基建上限核心计算逻辑。P0 应用的主数据源，等级跟随应用 |
| `ads_srsv_bi_ad_optimizer_template_target_data_pre_1` | **P0** | 主表的前置处理表，包含广告组汇总、优化师匹配、标准关联、淘汰判断等关键计算。与主表共同构成基建上限数据产品 |
| `ads_advertisement_adext_view` | **P1** | 广告扩展信息视图，提供书籍ID等维度。被 P0 表直接引用 |
| `ads_creation_ad_set_task_log_view` | **P1** | 创编任务-广告组关联日志，用于关联模板与广告组。P0 表拼接模板维度的关键桥梁 |
| `ads_creation_ad_set_task_view` | **P1** | 创编任务视图，提供任务与广告组映射。P0 表模板关联的数据源 |
| `ads_creation_template_view` | **P1** | 创编模板视图，提供模板名称。P0 表模板维度数据源 |
| `ads_bi_ad_cost_recharge_view` | **P1** | 广告花费充值宽表，在优化师匹配逻辑中用于判断近7天花费最高的优化师。P0 表优化师分配的关键依据 |
| `ads_bi_ad_new_user_value_ed` | **P1** | 新用户价值增量表，提供 D0 收入、新增用户数等核心指标。P0 表达标率计算的数据基础 |
| `ads_srsv_ads_marketing_plan_view` | **P1** | 营销计划视图（含测试排期、代号阶段、代号等级），用于 P0 表过滤 B 级书和禁投状态 |

## 特别说明

1. 本分析基于单一应用的「下游辐射上游」传导规则。若后续同一张表被其他 P0/P1 应用引用，等级取两者中较高者。
2. 3 张 DIM 表（`dim_srsv_ad_prduct_lang_rate`、`dim_srsv_ad_prduct_source_rate`、`DIM_ProductType`）在 starrocks/ 目录下无 DDL/DML 文件，可能为外部 ETL 加载或跨系统配置表，建议与数仓负责人确认其数据来源。
3. `dim_pub_code_mapping_dict` 为静态码表（INSERT VALUES），无上游依赖。
4. 书籍/内容域 ODS 表（17 张）通过 `dim_shuangwen_book_read_consume_info` 和 `dim_sv_series_hi` 间接服务于 P0 应用，等级为 P1 而非更低，因为 P0 应用依赖其产出的书籍代号和语言维度。
